# 口播音频 → Remotion 动效解说视频

把一条口播音频做成全屏 motion graphics 解说视频：观众听原声口播，画面用动效、emoji、logo、大数字帮助理解。示例成片：《Q2 财报周前瞻》1080×1080@30fps 3'08"。

## 流程总览（六步，第 3 步是硬闸门）

1. **转写** — 带时间戳转写音频，通读全文
2. **写动效计划** — 视觉系统 + 逐段分镜表 + 待确认点
3. **⛔ 给用户过目，等确认后再写代码** — 用户没点头前不搭工程。动效视频返工成本极高（改一版=重排场景+重渲染），把分歧消灭在计划阶段是本 skill 的流程合约
4. **搭 Remotion 工程** — 场景代码 + 时间轴对齐
5. **关键帧抽查** — 每个场景渲 1-3 帧静帧，肉眼过一遍再整片渲染
6. **整片渲染 + 交付** — MP4 拷回音频所在目录，汇报成片参数和分镜清单

## Step 1 转写

用本 skill 自带脚本（fsmn-vad 切段 + SenseVoice 逐段识别；依赖 `pip install funasr soundfile` 和 ffmpeg，首次运行自动从 ModelScope 下载模型）：

```bash
python3 scripts/transcribe_ts.py <音频文件> <输出transcript.json>
```

输出每段 `{start, end, text}`。**注意**：去过气口（剪掉呼吸停顿）的音频句间无停顿，VAD 会切出 60s+ 的长段——属正常，句级时间点用下一步的字符比例法解决。

通读全文，把 ASR 错字按正确写法记下来（品牌名、术语、数字单位最容易错），上屏文字一律用正确写法，数字以口播为准。

## Step 2 动效计划（交给用户看的文档）

先算好分镜切点：

```bash
python3 scripts/align_marks.py transcript.json "开场句开头" "第二段开头" ...
```

它对长段内的子串按字符比例估时（±1s，够用），输出秒数和 30fps 帧号。

计划文档结构（markdown 发在对话里）：
1. **视觉系统** — 引用品牌规范（见 references/design-system.md），说明底色/强调色/涨跌色/主体出场规范/文字纪律/转场
2. **分镜表** — 每行：编号 | 时间区间 | 口播内容概括 | 画面与动效描述。分镜按口播语义切段，一个场景讲一件事
3. **执行方案** — 工程位置、素材来源、渲染产出路径
4. **待确认点** — 至少要问清：**画面比例**（1:1 / 9:16 / 16:9，不同比例布局参数完全不同，中途改=每个场景重排，务必先定）；要不要字幕轨（默认不上整句字幕，只有画面关键词，但**默认在底部留 ~170px 字幕安全区**供用户后期加字幕，全部内容 bottom ≥ 175）；口播里提到高管/名人时，问是否要真人抠图出场

发出计划后**停下**，等用户回复。用户指定改哪个分镜就改哪个。

## Step 3-4 工程与场景

读 `references/remotion-workflow.md`（工程模板、组件清单、渲染命令）。写场景代码前读 `references/design-system.md`（品牌 tokens、动效语言、布局护栏、踩过的坑）。

核心约束（详见参考文档，这里是不能破的底线）：
- 每帧独立可计算：动画只依赖 `useCurrentFrame()` + interpolate/spring，不用 state/随机数（Remotion 并行乱序渲帧）
- 屏上只出现**关键词、数字、emoji、logo**，绝不整句上屏
- 时间轴帧号全部来自 transcript 对齐结果，写进 `timeline.js` 统一管理，场景内只引用常量

## Step 5 关键帧抽查

每个场景挑 1-3 个信息最满的帧渲静帧，作为图片查看：

```bash
npx remotion still src/index.jsx <CompId> out/stills/f<N>.png --frame=<N>
```

逐帧检查清单：文字溢出/换行、元素相互重叠（尤其顶部章节标签 vs 场景标题）、上一拍残影没退干净、emoji 是否自带违和底色块（方块底箭头类 emoji 一律换 SVG/无底色符号）、连线是否对准目标。发现问题改完**重渲同一帧**确认，再进入整片渲染。

## Step 6 渲染与交付

```bash
npx remotion render src/index.jsx <CompId> "out/<成片名>.mp4" --codec=h264 --crf=17
```

整片渲染放后台跑（约 10-20 分钟）。完成后：`ffprobe` 验证时长/分辨率/音轨都在 → 从成片抽 2 帧确认封装无误 → 拷贝到音频所在目录 → 汇报：成片路径、参数（分辨率/fps/时长/大小）、分镜清单、哪里可以微调。

## 设计铁律速查（完整版在 references/design-system.md）

- 深底 + 品牌母题常驻背景；高亮色**只给全片最关键的一两处信息**
- 涨=绿 `#2ebd85`，跌=红 `#f6465d`，固定不换
- 主体出场 = 官方 logo 圆角块 + `$TICKER` 药丸（logo 从 `assets.parqet.com/logos/symbol/<TICKER>?format=png&size=200` 拉）
- 数字一律 count-up 滚动
- **转场分层级**：只有大章节切换用径向擦除（前后各 10 帧 ≈0.67s），小节切换一律硬切——全片擦除 ≤10 次，否则转场吃掉太多时间
- **字幕安全区**：底部 ~170px 不放任何内容（贴底元素 bottom ≥ 175），给用户后期字幕留位
- **文字再少一点**：卡片只留一行主文案（副标题默认不加），chip ≤10 个字
- **入场过冲 ≤1.3**：Pop from>1.3 的过冲会让卡片入场瞬间放大出框；只有局部小元素（单个数字/图章）可用到 1.3
- 中文 PingFang 最粗只有 600，大数字用 `-apple-system` 800 补
